Rules

    Each solver must pass a regression suite consisting of small instances which caused at least one solver from the last evaluation to crash. The regression suite will be published latest at the end of march.

Starting in 2024: A solver is not allowed to use external binaries (including commercial MIP solvers) of unmodified third-party libraries. Open-source libraries are allowed.

    Starting in 2024, Solver Exit Codes:
    As in the incremental track, the solver must return one of the following exit codes:
    • 0   if it cannot find an optimal solution or prove unsatisfiability.
    • 10 if the instance contains only satisfiable hard clauses.
    • 20 if it finds that the hard clauses are unsatisfiable.
    • 30 if it finds an optimal solution.

Solver Development


As the yearly MaxSAT Evaluation drives MaxSAT solver development forward, we want to recommend some standardizations to the solvers to make them easier to use.
These points are open for discussion and no such instance will be included in the Benchmark set. This topic is also open for discussion in SAT solving, as some solvers, like kissat and CaDiCaL, can handle empty instances/clauses, while others cannot. These suggestions are in line with the mathematical logic of the Boolean satisfiability problem.

  • An empty instance should yield a weight "o 0", with an empty model line "v" and the status line "s OPTIMUM FOUND".
  • An empty hard clause should result in an unsatisfiable instance.
  • An empty soft clause should be unsatisfiable, but the instance can still be satisfiable.
  • A solver should accept weight 0 for a soft clause.

MaxSAT Fuzzer and Regression Suite

We further want to advertise the use of a regression suite which provides a set of small valid instances known to have caused issues in a previous solver.
Further, we plan to provide a MaxSAT fuzzing toolsuite (new release in a GitHub repository probably at the end of March) for exhaustive testing of MaxSAT solvers. This toolsuite compares the solution of your solver with any existing solver which complies to the rules of the MaxSAT Evaluation. It compares if the "o-value" is the same as the value provided by your model and if no other solver (e.g. a verified MaxSAT solver) can produce a better solution output with a correct model. It further is able to recognize solver crashes on valid instances (it does not produce empty clauses or instances). Then it minimizes the instance with a delta debugging algorithm to provide simple instances for debugging.
